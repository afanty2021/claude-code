## 设计思维框架

### 设计前理解
在编码前深入理解上下文并承诺大胆的美学方向：

1. **目的分析**: 解决什么问题？用户是谁？
2. **风格定位**: 选择极端方向：
   - 极简主义
   - 极繁主义混沌
   - 复古未来主义
   - 有机/自然风格
   - 奢华/精致
   - 俏皮/玩具风格
   - 编辑/杂志风格
   - 野兽派/原始
   - 装饰艺术/几何
   - 柔和/粉彩
   - 工业/实用主义

3. **约束识别**: 技术要求、框架、性能、可访问性
4. **差异化**: 是什么让这个设计令人难忘？

### 设计原则
- **意图性**: 无论强度如何，关键是有意识的设计
- **独特性**: 每个设计都应该不同，避免趋同
- **执行力**: 与美学愿景相匹配的实现复杂度

## 前端美学指南

### 核心关注点

#### 字体排印 (Typography)
- **字体选择**: 选择美丽、独特、有趣的字体
- **避免通用字**: 避免 Arial、Inter、Roboto 等通用字体
- **字体组合**: 将独特的展示字体与精致的正文字体配对
- **字符感**: 选择具有个性化和意外魅力的字体

#### 色彩与主题 (Color & Theme)
- **美学承诺**: 坚持统一的美学风格
- **CSS 变量**: 使用 CSS 变量保持一致性
- **主导色彩**: 主导色彩配合锐利强调色比温和均匀的调色板效果更好
- **主题变化**: 在浅色和深色主题间变化，不同设计使用不同调色板

#### 动效 (Motion)
- **CSS 优先**: HTML 使用纯 CSS 解决方案
- **React 支持**: React 使用 Motion 库
- **高影响时刻**: 一个精心编排的页面加载，配合错开显示的过渡效果
- **交互惊喜**: 使用滚动触发和悬停状态创造惊喜

#### 空间构成 (Spatial Composition)
- **意外布局**: 不对称、重叠、对角线流动
- **网格破坏**: 打破网格的元素
- **负空间**: 慷慨的负空间或受控的密度

#### 背景与视觉细节
- **氛围创造**: 创造氛围和深度，而不是默认使用纯色
- **纹理效果**: 添加符合整体美学的情境效果和纹理
- **创意形式**: 渐变网格、噪声纹理、几何图案、层次透明度、戏剧性阴影、装饰边框、自定义光标、颗粒叠加

### 禁忌模式
避免使用以下通用 AI 生成美学：
- 过度使用的字体系列 (Inter, Roboto, Arial, 系统字体)
- 陈词滥调的配色方案（特别是白色背景上的紫色渐变）
- 可预测的布局和组件模式
- 缺乏特定情境字符的通用设计

## 技术实现规范

### 支持的技术栈
- **HTML/CSS/JavaScript**: 原生 Web 技术
- **React**: 使用 Motion 库支持
- **Vue**: 响应式动画
- **其他框架**: 适配相应动画系统

### 实现要求
- **生产级质量**: 功能完整，性能优化
- **视觉冲击**: 令人难忘和视觉震撼
- **美学统一**: 具有清晰美学观点的统一性
- **细节精致**: 每个细节都经过精心完善

### 复杂度匹配
- **极繁设计**: 需要复杂代码，包含广泛动画和效果
- **极简设计**: 需要克制、精确和对间距、字体、微妙细节的仔细关注
- **优雅**: 来自于对愿景的良好执行

## 测试与质量

### 质量保证
- **视觉测试**: 设计在不同设备和浏览器的一致性
- **性能测试**: 动效和交互的性能优化

### 验证标准
- **独特性**: 与通用设计有明显区别
- **功能性**: 所有交互功能正常工作
- **美观性**: 符合既定美学标准

## 常见问题 (FAQ)

**Q: 如何选择合适的美学方向？**
A: 基于项目目的、目标受众和品牌特性选择，避免随机组合。

**Q: 确保设计独特性的方法？**
A: 避免跨代使用相同选择（如 Space Grotesk），每个设计都要有独特的视觉个性。

**Q: 如何平衡美学和性能？**
A: 根据美学愿景匹配实现复杂度，极繁设计接受更高的性能开销。

**Q: 支持哪些类型的动画？**
A: 优先 CSS 动画，复杂交互使用专门的动画库。
