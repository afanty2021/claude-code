# Learning Output Style Plugin

## 模块职责

提供学习导向的输出风格插件，通过会话开始钩子配置 Claude 以教育和学习为重点的响应模式，强调概念理解、知识传递和技能培养。

## 入口与启动

### 插件元数据
- **配置文件**: `.claude-plugin/plugin.json`
- **版本**: 1.0.0

### 核心组件结构
```
learning-output-style/
├── .claude-plugin/plugin.json              # 插件元数据
├── README.md                               # 插件文档
├── hooks/
│   ├── hooks.json                          # 钩子配置
│   └── hooks-handlers/
│       └── session-start.sh               # 会话开始处理器
└── CLAUDE.md                               # 模块文档
```

## 对外接口

### 钩子配置
```json
{
  "description": "Configures Claude to provide learning-oriented, educational responses",
  "hooks": {
    "SessionStart": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "${CLAUDE_PLUGIN_ROOT}/hooks-handlers/session-start.sh"
          }
        ]
      }
    ]
  }
}
```

### 触发时机
- **SessionStart**: 在每个新的 Claude 会话开始时自动触发
- 配置 Claude 的输出模式为学习导向风格

## 钩子实现机制

### 会话开始处理器
`hooks-handlers/session-start.sh` 负责在新会话开始时：

1. **自动初始化**: 会话启动时自动执行配置
2. **学习模式**: 设置 Claude 为教育导向的响应模式
3. **无感知操作**: 用户无需手动配置，自动生效

### 实现特点
- **轻量化**: 最小化的 shell 脚本实现
- **自动化**: 会话级别的自动配置
- **标准化**: 遵循 Claude Code 钩子标准

## 学习导向输出特征

### 教育性响应风格
启用此插件后，Claude 将提供：

#### 知识传递
- **概念解释**: 详细解释技术概念和原理
- **背景知识**: 提供相关的历史和理论背景
- **联系实际**: 将抽象概念与实际应用联系

#### 技能培养
- **逐步指导**: 分步骤展示解决问题的方法
- **思考过程**: 展示分析问题和推理的过程
- **最佳实践**: 分享行业最佳实践和经验

#### 互动学习
- **提问引导**: 通过提问引导用户思考
- **探索鼓励**: 鼓励用户尝试不同的解决方案
- **错误学习**: 从错误中学习和改进

#### 深度理解
- **原理解释**: 不仅告诉"怎么做"，还解释"为什么"
- **多样性**: 展示多种解决方法和各自的优劣
- **延伸思考**: 引导用户思考相关问题和扩展应用

### 适用场景
- **技术学习**: 学习新的编程语言、框架或工具
- **概念理解**: 理解复杂的计算机科学概念
- **问题解决**: 培养解决问题的思维和能力
- **代码学习**: 深入理解代码设计和实现原理

## 常见问题 (FAQ)

**Q: 学习导向风格会回答更慢吗？**
A: 可能会稍慢一些，因为包含更详细的解释和背景知识

**Q: 适合什么水平的用户？**
A: 适合所有水平的用户，系统会根据问题复杂度调整解释深度

**Q: 可以临时禁用学习风格吗？**
A: 可以在对话中明确要求简洁回答，临时覆盖学习模式

**Q: 与解释性风格有什么区别？**
A: 学习风格更强调教育和技能培养，解释性风格更侧重问题解释
