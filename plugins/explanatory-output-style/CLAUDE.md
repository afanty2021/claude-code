# Explanatory Output Style Plugin

## 模块职责

提供详细解释性的输出风格插件，通过会话开始钩子配置 Claude 的输出模式，使其提供更详细、教育性的响应。

## 入口与启动

### 插件元数据
- **配置文件**: `.claude-plugin/plugin.json`
- **版本**: 1.0.0

### 核心组件结构
```
explanatory-output-style/
├── .claude-plugin/plugin.json              # 插件元数据
├── README.md                               # 插件文档
├── hooks/
│   ├── hooks.json                          # 钩子配置
│   └── hooks-handlers/
│       └── session-start.sh               # 会话开始处理器
└── CLAUDE.md                               # 模块文档
```

## 对外接口

### 钩子配置
```json
{
  "description": "Configures Claude to provide detailed, explanatory responses",
  "hooks": {
    "SessionStart": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "${CLAUDE_PLUGIN_ROOT}/hooks-handlers/session-start.sh"
          }
        ]
      }
    ]
  }
}
```

### 触发时机
- **SessionStart**: 在每个新的 Claude 会话开始时自动触发
- 配置 Claude 的输出风格为解释性模式

## 钩子实现机制

### 会话开始处理器
`hooks-handlers/session-start.sh` 是一个简单的 shell 脚本，用于：

1. **自动执行**: 在会话开始时自动运行
2. **风格配置**: 设置 Claude 的输出偏好
3. **无干扰**: 不影响正常的对话流程

### 实现特点
- **轻量级**: 最小化的实现，专注于配置功能
- **可靠性**: 使用标准 shell 命令，兼容性好
- **透明性**: 用户无感知的背景配置

## 常见问题 (FAQ)

**Q: 如何禁用解释性输出风格？**
A: 可以通过卸载插件或在特定会话中明确要求简洁回答

**Q: 钩子会影响性能吗？**
A: 影响极小，钩子只在会话开始时执行一次

**Q: 可以与其他输出风格插件同时使用吗？**
A: 不建议，可能会导致冲突的行为

**Q: 如何验证插件是否正常工作？**
A: 开始新会话并观察 Claude 的响应风格是否更加详细
